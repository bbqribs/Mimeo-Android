Mimeo · Implementation handoff · v1

From visual design to a buildable spec.

This document converts the approved visual direction (Direction B · Paper & Ember as the canonical baseline, with Direction A retained as the dark-theme variant) into a concrete implementation contract for Android (Jetpack Compose) and the server-rendered web app. Tokens, component rules, mapping snippets, rollout, QA — no new product, no new features.

1 · Design tokens

All numeric values are unit-typed: dp on Android, px on web (1 dp ≈ 1 px at MDPI; the system scales). Token names are stable; theme files swap the values.

1.3 Accent variants

Single brand accent per theme. Do not introduce per-screen accents. The "Now playing" tint is derived from the same accent at low alpha. If we ever ship user-pickable accents, expose only these curated values:

  • Light: #C25B2E (ember, default), #3F7A52 (forest), #2B567A (slate blue), #7A3E8C (plum).
  • Dark: #B6A1FF (lilac, default), #8FB6FF (steel), #7FD1A8 (sage), #E8C26A (sand).

1.9 Elevation / shadow

Shadows are reserved for transient surfaces (sheets, snackbars, the floating "Jump to Now Playing" pill). All persistent surfaces use borders, not shadows. This keeps light mode looking like paper and dark mode looking flat instead of plasticky.

TokenValueUse
elev.0noneCards, drawer, app-bar

elev.30 12 32 rgba(0,0,0,.18)Bottom sheets, dialogs

1.10 Divider / border rules

  • Within a list: 1-px color.line.soft hairline, indented to start at the left edge of the row's text content (≈ 18 px from the screen edge, or 42 px when a drag handle is present).
  • Between sections: No divider — separate by section header type + spacing only.
  • Cards / containers: 1-px color.line all sides.
  • Now-playing card: 1-px color.accent all sides + 3-px left edge; no shadow.
  • App bar & sticky chip strip: 1-px color.line.soft bottom only.
  • Never combine a shadow and a border on the same element.

2 · Component rules

2.1 App shell & drawer

Layout

App bar height 56 dp. Title in type.title. Leading icon button is 44 × 44 dp tap target. Trailing action cluster is right-aligned, max 3 icons; overflow into more.

Drawer

Width 288 dp. Background color.surface (light) / #0E0E14 (dark). Branding header (logo + wordmark) at top, 24 dp padding. Sections in this fixed order, each preceded by a type.section header:

  1. Library: Inbox · Favorites · Archive · Bin (counts right-aligned)
  2. Listen: Up Next · Smart Queue · Bluesky
  3. Playlists: manual playlists, alphabetical, with item count
  4. Smart playlists: diamond glyph, with live result count
  5. Settings: pinned to bottom, separated by 1-px color.line top

Drawer row

48 dp height, 18 dp horizontal padding, 14 dp icon-label gap. Selected: color.accent.dim background, radius.card, label and icon in color.accent, weight 600. Counts in type.meta right-aligned.

States

Hover (web) / pressed (Android): color.surface.hi. Disabled (e.g. Bin when empty): label color.fg.3, count omitted, no hover effect.

2.2 Library item row

Layout (single column)

  • Vertical padding from density.row.padV; horizontal 18 dp.
  • Row 1 — title, type.row, color.fg, max 2 lines, ellipsize end.
  • Row 2 — meta, type.meta, color.fg.3: host · duration. If favourited, append a tiny star glyph in color.accent, 12 dp.
  • Optional Row 3 — progress bar 2 dp tall, max 240 dp wide, color.line.soft background, color.accent fill at 70 % opacity. Only render when read > 0.
  • Trailing — Play/Pause button (color.accent, 36 dp) and Overflow (color.fg.2, 36 dp).

Actions

  • Tap row: open Reader.
  • Tap Play: place item as Now Playing in the current Up Next session, start TTS. Does not clear Up Next.
  • Tap overflow: bottom sheet — Add to Up Next · Add to playlist · Favourite · Archive · Move to Bin.
  • Swipe-left: Archive (matches Material default; surfaced visually with color.line background reveal).

States

Now-playing: row tinted with color.now.tint, Play icon switches to Pause in color.accent. Pressed: row background color.surface.hi. Read 100 %: title in color.fg.2, no progress bar.

2.3 Playlist item row

Same skeleton as Library row, with the addition of a drag handle on the leading edge (manual playlists only) — 22 dp I.drag glyph in color.fg.4, 8 dp from screen edge. Long-press initiates reorder; haptic + 4 % scale feedback. Smart playlist rows do not show a handle.

2.4 Smart Queue row

Identical to Library row visually. Difference is in the surrounding chrome: a sticky filter bar above the list with search field (radius.card, color.surface bg) and chip strip (Newest · Oldest · Opened · Progress · All sources). Sections inside the list are time-bucketed (Today · This week · Earlier) using type.section headers and color.accent for the active bucket.

2.5 Up Next row & sections

Section order (top → bottom)

  1. History — items already played in this session. Rows rendered at opacity 0.55, dense density (override density.row.padV by −2). No drag handle.
  2. Earlier in queue — items skipped past. Dense, no drag handle.
  3. Now playing — anchored card (see below). Always present even at session start (collapsed empty state with prompt: "Pick something from Smart Queue or your Inbox.").
  4. Up Next — upcoming items. Default density. Drag handles enabled; reorder is the only interaction unique to this section.

Now-playing card

  • radius.card, color.now.tint background, 1-px color.accent border, 3-px left edge.
  • Top kicker: pulsing dot + "Now playing · 4:32 / 9:18" in type.section, color.accent.
  • Title: type.title, max 3 lines.
  • Meta: type.meta, host only.
  • Progress: 3-px bar, color.line track, color.accent fill.
  • Tapping the card opens the Reader, prefilled with the playing item.

Header actions

App-bar trailing icons: refresh (re-seed Up Next from Smart Queue, with confirmation dialog), add (queue picker sheet), more. The "Re-seed" copy ALSO appears as a labelled secondary button beneath the session-meta row, for first-use discoverability.

2.6 Reader / Locus

Layout

  • App bar with back arrow + favourite, archive, more.
  • Now-playing bridge — only when this article is the playing item: 40 dp band below the app bar, color.now.tint background, dot + "Now playing · time" + speed pill.
  • Article header: kicker (host name in type.section, color.accent), title type.display, byline meta in type.meta.
  • Body: type.body.read in color.fg. First paragraph italicised when it reads as a kicker/dek (heuristic: ≤ 200 chars, plain-text). Highlights from TTS sync render as inline color.accent.dim background.
  • Max content width 640 dp, centered.

Mini-player

Persistent across all screens except the Reader showing a different item (where the embedded bridge stands in). When reading a different item than what is playing, surface a floating "Jump to Now Playing" pill (elev.2) above the mini-player. Tapping it reroutes the Reader to the playing item; it does not modify Up Next.

Actions

Favourite ↔ heart fill toggle. Archive moves the item to Archive and snackbars "Archived · Undo". More opens: Mark as read · Share · Open in browser · Move to Bin.

2.7 Bluesky candidate browser

Hero context band

One-line privacy reminder, type.meta, color.fg.3: Live candidate links. Saving creates normal Mimeo items — nothing is harvested without you. The phrase "nothing is harvested without you" is bolded, color.fg.2. This band is required; it is the only way the user reconciles the live-feed metaphor with the no-auto-save policy.

Source selector card

radius.card, color.surface, 1-px color.line. Shows current source (Timeline / List / Feed / Account) and resolves to a list/picker on tap. Adding a source is two-step: pick kind, then enter handle/URL.

Status pill

Inline strip below selector. Shows scan state: dot + "Scanned N posts · M links · stop: max_age | rate_limit | manual". color.accent on color.accent.dim, radius.item.

Candidate card

  • One per scanned link. radius.card, color.surface, 1-px color.line.
  • Title type.row, URL type.meta truncated, byline author · @handle · age.
  • Status badge in the top-right corner: Saved (color.accent.dim bg, accent fg) or Unsaved (color.line.soft bg, color.fg.3 fg).
  • Action row: Open link · Open post · trailing Save primary button. When already saved, the button switches to a secondary ✓ Saved pill and tapping again opens an undo sheet (Undo · Open in Mimeo · Move to playlist).

Hard rules

  • Save is the only way an item enters Mimeo. There is no auto-save toggle, no implied save on scroll, no save-by-default.
  • Saving never modifies Up Next. To play, the user opens the saved item in Mimeo and presses Play, or adds it from a row overflow.
  • Scanning never decrements a quota or surfaces "we crawled X" telemetry — the band's "Scanned 9 posts" is a session-local counter, not a server stat.

2.8 Settings section

Settings are organised into groups, each rendered as a single rounded card (radius.card, color.surface, 1-px color.line) with rows separated by 1-px color.line.soft hairlines. Each row is 56 dp:

  • Leading: 32-dp tinted icon — color.accent.dim bg, color.accent glyph. Destructive rows use color.danger on a 12 % danger background.
  • Center: Label type.body 14.5/600, value type.meta below.
  • Trailing: Chevron, toggle, or chip-strip (e.g. Connection Mode: Local / LAN / Remote).

Group order: Connection · Sources · Reading & playback · Privacy. Connection group includes a tappable monospace endpoint, "Test connection" primary, "Diagnostics" secondary, and a green status line.

2.9 Dialog

Centred dialog

Width 320–360 dp, radius.sheet, color.surface, elev.3. Title type.title, body type.body, button row right-aligned: secondary (text only) + primary (filled). Destructive primary uses color.danger.

Bottom sheet

Used for row overflow and source pickers on mobile. radius.sheet top corners, drag handle 36 × 4 dp color.fg.4. Each action row 48 dp, 14 dp icon-label gap.

2.11 Empty / loading / error states

Empty

Centred 48-dp circular icon (color.surface bg, color.fg.3 glyph), type.title headline, type.body 1-line explanation, no illustration. Optional secondary action below.

Loading

  • List: 5 skeleton rows — title bar (60 % width × 12 dp), meta bar (40 % × 8 dp), color.line.soft, 1.4 s shimmer.
  • Reader: kicker bar + 3-line title bar + 6 paragraph bars; same shimmer.
  • Bluesky scan: the status pill animates a striped fill across the dot row to indicate active scan; cards shimmer in as they arrive.

Error

Inline at the top of the affected list: color.danger 12 % bg, 1-px color.danger border, type.body message, secondary "Retry" outlined button. Never modal. Never dismisses on tap-outside.

3.6 Settings exposure

Three switches drive the entire visual surface. They live under Settings → Reading & playback:

  • Theme — System (default) · Light · Dark. Persisted in DataStore; flows through MimeoTheme.
  • Density — Compact · Default · Comfortable. Default for new installs. Affects all ArticleRow instances and section spacing.
  • Reader font — System serif · Source Serif 4 · Spectral · Crimson Pro. Custom faces are bundled, not downloaded.

None of these are time-of-day or motion-detected. They are explicit user choices. Do not auto-switch on schedule.

5 · Rollout plan

Goal: ship the visual change without freezing feature work. Each phase below is independently shippable behind a single feature flag (theme_v2) on Android and a build-time CSS swap on web.

Phase 0 — Foundation ~1 week

  • Land theme/Color.kt, theme/Type.kt, theme/Shapes.kt, theme/Spacing.kt, plus MimeoTheme wrapper.
  • Bundle Source Serif 4 + Inter + JetBrains Mono as Android font resources.
  • Stand up tokens.css on web. Both stylesheets coexist; old screens still consume the old tokens.
  • Wire Settings → Theme + Density + Reader font preference to DataStore / cookie. No UI yet — just the storage.

Phase 1 — Shared components ~1.5 weeks

  • Build :ui-core composables in this order (each replaces an existing helper):
    1. MimeoAppBar, MimeoButton, MimeoChip, MimeoIcon, EmptyState, SkeletonRow.
    2. ArticleRow (variant flag for Library / Smart Queue / Up Next / History).
    3. NowPlayingCard + MiniPlayer + Jump pill.
    4. SettingsGroup + SettingsRow.
    5. CandidateCard.
  • Web: equivalent partials (_row.html, _now_card.html, _mini_player.html) consuming tokens.css.
  • Snapshot tests for each component in light + dark + each density.

Phase 2 — Screens, in order ~3 weeks

Convert one screen per PR. Each PR ships disabled by default; QA flips the flag for that screen only.

  1. Drawer + App shell — sets the chrome.
  2. Inbox / Library — proves ArticleRow at scale.
  3. Up Next — proves NowPlayingCard, drag handles, reorder.
  4. Reader / Locus — proves serif body, Now-playing bridge, Jump pill.
  5. Smart Queue — proves filter strip + sticky search.

6 · Design QA checklist

Tokens & theming

  • Light and dark themes produce no contrast-ratio failures on body text (≥ 4.5:1) and large text (≥ 3:1).
  • System theme switch (Android quick settings) flips the app within 1 frame, no flicker.
  • No screen contains a raw hex value — grep #[0-9A-F]{6} against the screens module returns nothing.
  • Custom fonts load locally; airplane-mode launch shows correct typography on all screens.

Components

  • ArticleRow truncates at exactly 2 lines, never 3, regardless of language.
  • Tap targets are ≥ 44 dp on every interactive element including icon buttons.
  • Drag handle on Up Next + manual playlists ONLY. Smart playlists and History have no handle.
  • Now-playing card uses the 3-px left edge; no shadow.
  • Mini-player persists across all non-reader screens; "Jump to Now Playing" pill appears only when the open Reader item differs from the playing item.

Up Next semantics

  • Re-seeding Up Next from Smart Queue prompts a confirmation dialog if the current Up Next has unplayed items.
  • Playing an item from Inbox/Library inserts it as Now Playing without clearing existing Up Next entries.
  • History items render at opacity 0.55 and are not interactive for queue actions.

Bluesky semantics — non-negotiable

  • The hero context band is rendered above the source selector on every entry to the Bluesky screen.
  • Saving is the only path that creates a Mimeo item from Bluesky. No "auto-save" toggle exists in code or UI.
  • Saving never modifies Up Next. Verify by tracing: Save → Inbox + only.
  • Scanning state ("Scanned N posts · M links") is local to the session, not persisted server-side.

Privacy & diagnostics

  • Default crash reports contain no article titles, URLs, domains, or source identities. Verify by inspecting an actual report payload.
  • Connectivity events log mode switches and timeouts only; no URL fields.
  • "Playback session length" toggle is OFF by default.
  • The Privacy hero "What never leaves the device" panel matches the actual telemetry behaviour line-for-line.

Compose feasibility

  • No screen requires custom Modifier.layout or SubcomposeLayout — every screen is composable with stock Column / LazyColumn / Row.
  • No screen depends on BlurEffect or other API ≥ 31. Backwards-compat is preserved.
  • Reorderable Up Next uses androidx.compose.foundation.lazy.LazyColumn + a community reorder helper, not a custom drag system.

Web feasibility

  • Server-rendered HTML for any screen is < 80 KB unzipped.
  • JavaScript is required only for the mini-player and reorder; the rest of every page works with JS disabled.
  • First paint uses the server-set theme cookie; no FOUC between system-default and user preference.
