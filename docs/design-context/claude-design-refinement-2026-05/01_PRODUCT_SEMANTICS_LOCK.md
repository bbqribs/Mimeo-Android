# Product Semantics Lock (Non-Negotiable)

1. Up Next is the playback session/target.
2. Smart Queue is a playlist-like source.
3. Bluesky scan is live candidate browsing.
4. Bluesky save creates a normal Mimeo item.
5. Bluesky scan/save must not auto-queue.
6. Privacy/diagnostics must not collect reading data by default.
7. Android is the primary daily-use client; web harmonises later.
8. Operator/admin screens are not the visual target.

## Additional guardrails
- Explicit user actions remain explicit (save, queue, move, archive, add/remove).
- Manual playlists and smart playlists stay distinct concepts.
- No hidden automation from candidate browsing into queue or playlist state.
