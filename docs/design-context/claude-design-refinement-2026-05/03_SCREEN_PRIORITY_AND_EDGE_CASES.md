# Screen Priority And Edge Cases

Limit the refinement pass to these screens only.

## 1) Inbox / Library
Request edge-case treatment for:
- Long titles and metadata wrap
- Empty state clarity
- Compact/default/comfortable density
- Large font scale
- Narrow width
- Light/dark contrast
- Row action crowding

## 2) Up Next
Request edge-case treatment for:
- Long titles across History / Earlier / Now Playing / Up Next sections
- Empty state and zero-session transitions
- Compact/default/comfortable density
- Large font scale
- Narrow width
- Light/dark contrast
- Row action crowding around reorder/overflow controls

## 3) Smart Queue
Request edge-case treatment for:
- Long source titles and queue summaries
- Empty state
- Compact/default/comfortable density
- Large font scale
- Narrow width
- Light/dark contrast
- Row action crowding

## 4) Reader / Locus
Request edge-case treatment for:
- Long titles and byline/source metadata
- Empty/unavailable content state
- Compact/default/comfortable density (controls + meta rows)
- Large font scale
- Narrow width
- Light/dark contrast for body text and controls
- Row action crowding in top/bottom action regions

## 5) Bluesky Candidate Browser
Request edge-case treatment for:
- Long candidate titles/handles
- Empty results / no candidates states
- Compact/default/comfortable density
- Large font scale
- Narrow width
- Light/dark contrast
- Row action crowding for save/pin/open actions

## 6) Settings / Privacy & diagnostics
Request edge-case treatment for:
- Long labels/values/tokens and diagnostics text blocks
- Empty/unknown diagnostics state
- Compact/default/comfortable density
- Large font scale
- Narrow width
- Light/dark contrast
- Row action crowding for test/copy/reset actions
