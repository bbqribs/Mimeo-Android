# Mimeo Visual Redesign Brief (May 2026)

## Purpose
This package gives Claude Design a compact, implementation-grounded context for a visual redesign pass.

## Product framing
- Mimeo is a read-later plus text-to-speech (TTS) product.
- Android is the primary daily-use client.
- Web is secondary, but should visually harmonise with Android for shared product concepts.

## Redesign goal
- Deliver an improved visual direction and interaction polish for existing product surfaces.
- Preserve current product semantics and behavior contracts.
- This is not a backend redesign or product-model rewrite.

## Scope guardrails
- Keep current queue, playlist, and save semantics intact.
- Keep explicit user actions explicit (save, queue, move, archive, etc.).
- Prioritise feasible UI patterns for Jetpack Compose (Android) and simple server-rendered web.

## Expected design output
- Updated visual language (hierarchy, spacing, typography, density, states, feedback).
- Clearer cross-surface continuity between Inbox/Library, Locus/Reader, Up Next, and playlist-related views.
- Harmonised but platform-appropriate Android and web presentation.
